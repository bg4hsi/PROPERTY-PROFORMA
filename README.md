# 投资测算 · 房地产项目投资利润测算应用

可直接运行的 Next.js MVP，用于房地产/商业综合体项目的投资测算。应用内置示例数据，支持实时计算、多个本地方案、给政府面积成本、自持业态经营回报、Excel 导入导出和打印为 PDF。

## 启动

需要 Node.js 18.17 或更高版本。

```bash
npm install
npm run dev
```

浏览器访问 `http://localhost:3000`。生产环境运行：

```bash
npm run build
npm start
```

## 主要功能

- 项目基础设置：增值税率、管理费率、销售费率、股东计息利率、交付月、试营业月数、四星/五星酒店房价、商业租金、入住/出租率和年经营成本占比集中管理；推演截止月位于销售与回款逻辑设置中。
- Excel 式可编辑主表：修改录入格后，全部结果即时刷新。
- 业态行：新增、复制、删除；自动生成项目合计。
- 业态分类：销售、自持酒店、自持商业、给政府、其他自持；分类决定进入销售回款、给政府成本或自持经营回报模型。给政府面积由分类自动计算。
- 政府配套：黄色高亮政府业态，展示政府面积、占比及成本。
- 自持回报：年收入、成本、净现金流、持有期、NPV、IRR，可选择计入项目收入。
- 全周期现金流图：按月展示新增销售、实际回款、项目支出，以及累计销售、回款、支出和净现金流；鼠标悬停任意月份可查看完整单月与累计明细。
- 逐业态回款引擎：配置首售月、交付月、总套数、月去化、首付、月回款和现房尾款分期，并输出截点去化、销售额及已回款。
- 方案管理：新建、复制、重命名、删除，自动保存在浏览器本地。
- Excel：导入“业态录入”工作表；导出录入、结果和项目参数三个工作表。
- PDF：点击“导出 PDF”后，在系统打印窗口选择“另存为 PDF”。这样可完整保留中文字体。

## 核心公式

所有公式集中在 `src/lib/calculationEngine.ts`，UI 中不包含业务计算。

| 指标 | 公式 |
|---|---|
| 销售面积（平方米） | `建筑面积 × 得房率`；得房率按百分比录入 |
| 销售总额（万元） | `销售面积 × 销售单价 ÷ 10000` |
| 建安成本-销售及自持 | `(建筑面积 - 政府面积) × 单方成本 ÷ 10000` |
| 建安成本-政府 | `政府面积 × 单方成本 ÷ 10000` |
| 建安成本-合计 | `销售及自持成本 + 政府成本` |
| 单方分摊地价（元/㎡） | 自动双模型：存在“给政府”业态时，沿用给政府建安成本分摊模型；不存在时，土地总价进入项目成本，可售物业按销售面积、自持物业按建筑面积分摊，地库/车位不参与 |
| 管理费用 | 可售业态：`货值 × 管理费率`；不可售业态：`建安成本合计 × 管理费率`，可手动覆盖 |
| 销售费用 | 可售业态：`销售总额 × 销售费率`；不可售业态为 `0`，可售业态可手动覆盖 |
| 增值税 | 销售价格为含税价：`销售总额 ÷ (1 + 增值税率) × 增值税率` |
| 股东计息 | 每月按 `MAX(0, -当月累计净现金流) × 股东年利率 ÷ 12` 计算，36 个月合计后按各业态建安成本占比分摊；计入项目总成本和利润测算，但不计入现金流 |
| 业态净利润 | 仅销售业态计算：`销售总额 - 建安成本合计 - 单方分摊地价 × 销售面积 ÷ 10000 - 管理费用 - 销售费用 - 增值税 - 股东计息`；非销售业态不计算 |
| 单方成本（含销管费） | `(建安成本合计 + 管理费用 + 销售费用) × 10000 ÷ 建筑面积` |
| 单方利润（元/㎡） | 仅销售业态计算：`业态净利润 × 10000 ÷ 销售面积` |
| 项目回报率 | `项目净利润 ÷ 项目总成本` |
| 不计经营回报净利润 | `销售收入 - 项目总成本` |
| 计入经营回报净利润 | `销售收入 + 自持累计经营回报 - 项目总成本` |
| 两种回报率 | 对应口径净利润分别除以项目总成本，两组数据始终同时展示 |
| 建设期资金峰值 | 交付月以内累计净现金流最低值的绝对值 |
| 年净现金流 | `年租金收入 + 年经营收入 - 年经营成本` |
| 年经营成本 | `(年租金收入 + 年经营收入) × 年经营成本占比` |
| 累计回报 | `年净现金流 × 持有年限` |
| 投资回收期（年） | 政府模型：`建安成本合计 ÷ 年净现金流`；土地总价模型：`（建安成本合计 + 分摊土地成本）÷ 年净现金流`；年净现金流小于或等于 0 时不计算 |
| NPV | 初始现金流为负投资成本（土地总价模型包含分摊土地成本），之后每年为年净现金流，按折现率折现 |
| IRR | 对上述现金流用二分法求净现值为 0 的收益率 |

现金流推演默认采用 36 个月周期。销售与回款采用各业态的回款逻辑配置，尚未配置的历史销售业态会自动生成兼容参数。建安成本的 75% 从第 1 月至交付月按月平均支付，剩余 25% 在交付后的 6 个月内平均支付。土地总价模型下，土地总价在第 1 月进入现金流支出。模型位于 `calculateCollectionSchedule` 和 `calculateCashFlowProjection`。

配置逐业态回款逻辑后，现金流图的销售和回款会改用实际批次模型：每月按“月去化套数”形成一批销售，销售当月收取首付款，交付前每月按“月回款比例”收款，未收余额从交付月（现房则从销售次月）开始按尾款分期月数平均收取。每笔累计回款均以该批销售额为上限，避免重复回款。

推演面板中的管理费率、销售费率、增值税率和交付月只作用于当前模拟，不修改项目基础参数。

月度支出中，管理费用从第 1 月至交付月平均分摊；销售费用按销售与回款引擎生成的“当月销售额 × 销售费率”计算。不可售业态不产生销售费用。

勾选“自持业态经营回报计入项目总收入”后，自持业态月度经营净现金流从 `交付月 + 试营业月数 + 1` 月开始进入资金曲线；月度金额为各自持业态年净现金流合计除以 12。

建筑面积视为包含政府面积，因此销售及自持成本先扣除政府面积，随后单独计入政府成本，避免同一面积重复计价。

自持酒店年经营收入按 `客房数 × 对应星级单客房平均每晚房价 × 酒店入住率 × 365 ÷ 10000` 计算；业态名称含“五星/五星级/5星”时套用五星价格，含“四星/四星级/4星”或未注明星级时套用四星价格。默认客房数按建筑面积每 50 平方米一间估算，可在自持回报表修改。自持商业年租金收入按 `建筑面积 × 每平方米月租金 × 商业出租率 × 12 ÷ 10000` 计算。

无论采用政府模型还是土地总价模型，名称包含“地库、地下车库、地下室、车位、停车”的业态都不进入自持业态经营回报，也不参与土地总价分摊。

## Excel 导入格式

推荐使用应用导出的 Excel 作为模板。导入时优先读取“业态录入”工作表，否则读取第一个工作表。必要列为“业态”；其他支持列：类型、建筑面积（平方米）、给政府面积（平方米）、得房率、销售面积（平方米）、销售单价（元/平方米）、单方成本（元/平方米）、管理费用覆盖（万元）、销售费用覆盖（万元）。旧字段“分配政府面积（平方米）”仍兼容。有“得房率”时销售面积会由公式重算；没有该列的旧表格则使用销售面积反推得房率。

## 目录结构

```text
src/
  app/                    Next.js 页面与全局样式
  components/             主表、项目参数、分摊和经营回报组件
  lib/
    calculationEngine.ts  集中式计算引擎
    excel.ts              Excel 导入导出
    sampleData.ts         内置示例数据
  store/                  Zustand 本地持久化状态
  types/                  TypeScript 业务类型
```

## 货币口径

引擎内部及项目总收入、总成本、净利润统一使用万元人民币；销售单价与单方成本按元/平方米录入。
